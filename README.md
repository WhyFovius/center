## 🎯 О проекте

**Zero Day** — это полноценный интерактивный симулятор кибербезопасности, разработанный для обучения сотрудников организаций основам цифровой гигиены через практический опыт. Вместо скучных презентаций и теоретических лекций пользователи погружаются в реалистичные сценарии кибератак, где каждое их решение имеет последствия.

### 💡 Ключевая идея

> *"Лучший способ понять опасность фишинга — попасться на фишинговую приманку в безопасной симуляции."*

Пользователь играет роль рядового сотрудника, который получает письма, сообщения и сталкивается с ситуациями, имитирующими реальные кибератаки. На каждом шаге нужно принять решение: открыть вложение? Перейти по ссылке? Подключиться к открытой Wi-Fi сети? Система оценивает выбор, показывает последствия и объясняет, что было сделано правильно или ошибочно.

### 🏗️ Что внутри

- **Полноценная имитация ОС** — рабочий стол, мобильное устройство, приложения (почта, браузер, мессенджер, терминал, настройки)
- **4 сюжетные миссии** с нарастающей сложностью: от базового фишинга до сложных многоэтапных атак
- **8+ шагов в каждом сценарии** с множеством вариантов решений
- **Система прогресса** — уровень безопасности, репутация, лиги, лидерборд
- **Генерация сертификатов** — подтверждение прохождения обучения
- **Адаптивный дизайн** — работает на десктопе и мобильных устройствах

### 🖥️ Архитектура Website

**Клиентская часть:**
- **React 19** — основная UI-библиотека с concurrent features
- **TypeScript** — строгая типизация всех компонентов и данных
- **Vite** — быстрый сборщик с HMR (Hot Module Replacement)
- **Zustand** — легковесное управление состоянием приложения
- **Framer Motion** — плавные анимации переходов между экранами
- **Tailwind CSS v4** — утилитарный CSS-фреймворк для стилизации

**Серверная часть:**
- **FastAPI (Python)** — высокопроизводительный REST API
- **SQLAlchemy 2.0** — ORM для работы с базой данных
- **Pydantic** — валидация и сериализация данных
- **JWT (python-jose)** — аутентификация через токены
- **bcrypt** — хеширование паролей
- **SQLite** — локальная база данных для разработки
- **PostgreSQL** — продакшен база данных
- **Redis** — кэширование лидербордов и сессий

**Коммуникация:**
- **HTTP REST API** — основные запросы (аутентификация, сценарии, лидерборд)
- **WebSocket** — обновление прогресса в реальном времени

#### 🔄 Система сценариев

Каждый сценарий представляет собой **дерево решений**, где пользовательский выбор определяет развитие истории:

```
Начало сценария
       │
       ▼
  ┌─────────┐
  │ Сценарий│  ←Narrative: Описание ситуации + эмоциональное давление
  │    1    │  ←Pressure: "СРОЧНО", "Ваш аккаунт заблокирован"
  └────┬────┘
       │
       ▼
  ┌─────────────────────┐
  │    4 варианта       │  ←A: Правильный (удалить письмо)
  │    решения          │  ←B: Нейтральный (проигнорировать)
  └────┬────────────────┘  ←C: Рискованный (проверить ссылку)
       │                   ←D: Опасный (кликнуть по ссылке)
       ▼
  ┌─────────────────────┐
  │    Последствия      │  ←Визуальная обратная связь
  │   + объяснение      │  ←Изменение security score
  └────┬────────────────┘  ←Образовательный блок (OWASP/CWE)
       │
       ▼
  Следующий шаг сценария (адаптируется под предыдущий выбор)
```

#### Security Level (Уровень безопасности)

```
0% ─────────────────────────────────────────── 100%
│                                              │
🔓 Критическая угроза                    🛡️ Абсолютная защита
```

- **0-25%** 🔴 — Критический уровень. Пользователь совершает множество ошибок
- **26-50%** 🟠 — Низкий уровень. Есть значительные пробелы в знаниях
- **51-75%** 🟡 — Средний уровень. Базовая грамотность присутствует
- **76-90%** 🟢 — Хороший уровень. Осознанные решения в большинстве ситуаций
- **91-100%** ⭐ — Отличный уровень. Экспертная бдительность

### 💡 Система подсказок

Каждая задача имеет **3 уровня подсказок** с прогрессивным раскрытием информации:

1. **Уровень 1** — Лёгкий намёк
   - *«Обратите внимание на адрес отправителя...»*
2. **Уровень 2** — Конкретная подсказка
   - *«Домен отправителя не совпадает с официальным доменом компании»*
3. **Уровень 3** — Прямое указание
   - *«Это фишинговое письмо. Правильное действие — удалить его и сообщить в IT»*

### 🏅 Лидерборд

Таблица лидеров с сортировкой по репутации:
- Кэшируется в Redis для производительности
- Автоматически обновляется после каждой попытки

## 🚀 Быстрый старт

### Предварительные требования

- **Node.js** 20+ и npm 10+
- **Python** 3.12+
- **Git** для клонирования репозитория

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/WhyFovius/center.git
cd center
```

### 2. Настройка фронтенда

```bash
# Перейдите в директорию фронтенда
cd frontend

# Установите зависимости
npm install

# Создайте файл переменных окружения
cp .env.example .env
```

### 3. Настройка бэкенда

```bash
# Перейдите в директорию бэкенда
cd ../backend

# Установите зависимости
pip install -r requirements.txt

# Создайте файл переменных окружения
cp .env.example .env
```

Файл `.env` уже настроен для локальной разработки с SQLite.

### 4. Запуск в режиме разработки

**Терминал 1 — Бэкенд:**
```bash
cd backend
set PYTHONPATH=<путь_к_корню_проекта>
uvicorn app.main:app --reload --port 8000
```

**Терминал 2 — Фронтенд:**
```bash
cd frontend
npm run dev
```

### 5. Откройте в браузере

- 🌐 **Фронтенд:** http://localhost:5173
- 🔧 **Бэкенд API:** http://localhost:8000
- 📚 **API Docs (Swagger):** http://localhost:8000/docs
- 📖 **API Docs (ReDoc):** http://localhost:8000/redoc

## 🚀 Деплой на Vercel

### Предварительные требования

1. Аккаунт на [Vercel](https://vercel.com)
2. Установленный [Vercel CLI](https://vercel.com/docs/cli) (опционально)
3. Репозиторий на GitHub

### Способ 1: Через Vercel Dashboard (рекомендуется)

1. **Подключите репозиторий:**
   - Перейдите на https://vercel.com/new
   - Импортируйте ваш GitHub репозиторий
   - Укажите Root Directory: `frontend`

2. **Настройте переменные окружения:**

   | Переменная | Значение | Описание |
   |------------|----------|----------|
   | `DATABASE_URL` | `postgresql://user:pass@host/db` | URL PostgreSQL БД |
   | `SECRET_KEY` | случайная строка | Ключ JWT |
   | `REDIS_URL` | `redis://host:port` | URL Redis (опционально) |

3. **Нажмите Deploy** 🚀

### Способ 2: Через Vercel CLI

```bash
# Установите Vercel CLI
npm i -g vercel

# Авторизуйтесь
vercel login

# Задеплоййте из корня проекта
vercel

# Для продакшен деплоя
vercel --prod
```

### Подключение PostgreSQL (Production)

Для продакшен-развертывания бэкенда рекомендуется использовать:

- **Supabase** — бесплатный PostgreSQL хостинг: https://supabase.com
- **Neon** — Serverless PostgreSQL: https://neon.tech

Обновите `DATABASE_URL` в переменных окружения:
```env
DATABASE_URL=postgresql://user:password@host:5432/zeroday
```

### Подключение Redis (Production)

Для кэширования лидербордов в продакшене:

- **Upstash** — Serverless Redis (бесплатный план): https://upstash.com

Обновите `REDIS_URL`:
```env
REDIS_URL=rediss://default:password@host:6379
```

## 🔧 Решение проблем

### Частые ошибки

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `ModuleNotFoundError: No module named 'backend'` | PYTHONPATH не установлен | `set PYTHONPATH=<путь_к_корню>` |
| `CORS Error` | Неправильные CORS настройки | Проверьте `cors_origins` в config |
| `Database locked` | SQLite блокировка | Закройте другие процессы |
| `WebSocket disconnect` | Сервер недоступен | Проверьте `VITE_API_URL` |
| `JWT expired` | Токен истёк | Перезайдите в систему |
